üìñ Overview

Synaplan AI Support Chat is a powerful WordPress plugin that seamlessly integrates intelligent AI-powered chat functionality into your WordPress website. The plugin features a comprehensive step-by-step wizard that guides you through the setup process, including user registration, configuration, and widget integration.

üéâ Automatic Account Creation: This wizard automatically creates a free account on www.synaplan.com during the setup process!

The plugin provides instant AI-powered customer support with knowledge base integration, allowing you to upload documents and train the AI to answer questions specific to your business.

üíæ Installation

  2. Navigate to Plugins ‚Üí Add New

üöÄ Quick Start

Getting started with Synaplan AI Support Chat is easy. Follow these simple steps:

  5. Your chat widget will be automatically added to your site!

üßô Setup Wizard Guide

The Setup Wizard is the heart of the Synaplan plugin. It's designed to guide you through the entire configuration process in just 4 simple steps. Here's a detailed walkthrough:

1Account Creation

What Happens in This Step:

This step creates your Synaplan account and validates your credentials. Your account will be instantly activated without requiring email verification.

Required Information:

  • Email Address: A valid email address (pre-filled with your WordPress admin email)
  • Password: Minimum 6 characters, must include numbers and special characters (e.g., !@#$%^&*)
  • Website Language: Automatically detected from your WordPress settings, but you can change it
  • Terms Agreement: You must accept the Terms of Service and Privacy Policy

Optional Settings:

  • Debug Mode: Enable this if you're a developer and want to see detailed console logs of data collection, sanitization, and API transfers

Technical Process:

1. User enters credentials
2. Email validation (sanitize_email)
3. Password strength check (6+ chars, numbers, special chars)
4. Data saved to transient (temporary WordPress storage)
5. Wizard advances to Step 2
‚ö†Ô∏è Password Requirements: Your password must be at least 6 characters long and include both numbers AND special characters. Example: MyPass123!

2Configure Your Chat Widget

What Happens in This Step:

Customize how your AI assistant will interact with your website visitors. This is where you define the personality and appearance of your chat widget.

Configuration Options:

  • Welcome Message: The first message visitors see when they open the chat (default: "Hello! How can I help you today?")
  • AI Assistant Type: Choose from predefined prompt templates:
    • General Support Assistant
    • Technical Support Specialist
    • Sales Assistant
    • Customer Service Agent
  • Widget Color: Pick a color that matches your website's brand (color picker provided)
  • Widget Position: Choose where the chat button appears:
    • Bottom Right (most common)
    • Bottom Left
    • Bottom Center

Technical Process:

1. User selects preferences
2. Intro message sanitized (sanitize_textarea_field)
3. Color validated (sanitize_hex_color)
4. Settings saved to transient
5. Wizard advances to Step 3

3Add Knowledge Base (Optional)

What Happens in This Step:

Upload documents to train your AI assistant with information specific to your business. This step is optional but highly recommended for better, more accurate responses.

File Upload Specifications:

Specification Details
Supported Formats PDF, DOC, DOCX, TXT
Maximum File Size 10MB per file
Maximum Files 5 files per wizard session
Total Upload Limit 50MB per session

Upload Methods:

  • Drag & Drop: Drag files from your computer directly to the upload area
  • File Browser: Click "Select Files" to browse your computer
  • Skip: Check "Skip this step" if you want to add files later

Technical Process:

1. User selects/drops files
2. File type validation (MIME type check)
3. File size validation (max 10MB per file)
4. WordPress media library upload (wp_handle_upload)
5. Attachment created (wp_insert_attachment)
6. Files stored temporarily in media library
7. File references saved to transient
8. Wizard advances to Step 4
üí° Pro Tip: Upload FAQs, product manuals, policy documents, or any content that will help the AI answer customer questions more accurately.

4Review & Complete

What Happens in This Step:

Review all your settings before finalizing the setup. Once you click "Complete Setup", the wizard will create your account, configure your widget, and process your uploaded files.

Review Summary Includes:

  • Email address
  • Website language
  • Welcome message
  • AI assistant type
  • Widget color
  • Widget position

What Happens When You Click "Complete Setup":

1. All wizard data collected from transient
2. API call to Synaplan backend:
   - Create user account
   - Verify website URL
   - Generate API key
   - Create widget configuration
   - Process uploaded files (vectorization)
   - Configure AI prompt with file search
‚¨á
3. API key and user ID saved to WordPress options
4. Widget configuration saved to database
5. Setup marked as completed
6. Temporary files cleaned up from media library
7. Wizard data deleted from transient
8. Redirect to dashboard
üéâ Setup Complete! Your chat widget is now active on your website. You can customize it further from the Settings page.

Wizard Data Flow & Security

The wizard uses WordPress transients (temporary storage) to store your data between steps. This ensures:

  • Security: Data is tied to your user session
  • Privacy: Data expires after 1 hour if wizard is abandoned
  • Reliability: You can go back and change previous steps
  • Clean Storage: All temporary data is deleted after completion

Error Handling

The wizard includes comprehensive error handling at each step:

  • Email validation errors
  • Password strength warnings
  • File upload errors (size, type, server issues)
  • API connection failures
  • Network timeouts

All errors are displayed clearly with suggestions on how to fix them.

‚öôÔ∏è Technical Details

Architecture Overview

The plugin follows WordPress best practices and is built with the following architecture:

Core Classes:

  • Synaplan_WP_Core - Main plugin initialization and core functionality
  • Synaplan_WP_Admin - Admin interface and menu management
  • Synaplan_WP_Wizard - Setup wizard logic and step processing
  • Synaplan_WP_API - API communication with Synaplan backend
  • Synaplan_WP_Widget - Frontend widget rendering
  • Synaplan_WP_REST_API - REST API endpoints for widget

Wizard State Management

The wizard uses WordPress transients with a 1-hour expiration for storing step data: 

// Step data storage
set_transient('synaplan_wizard_data_' . $user_id, $data, HOUR_IN_SECONDS);

// Step data retrieval
$wizard_data = get_transient('synaplan_wizard_data_' . $user_id);

Data Sanitization

All user inputs are sanitized using WordPress functions:

  • sanitize_email() - Email addresses
  • sanitize_text_field() - Single-line text
  • sanitize_textarea_field() - Multi-line text
  • sanitize_hex_color() - Color values
  • sanitize_file_name() - File names
  • rest_sanitize_boolean() - Boolean values

File Upload Security

File uploads are handled securely:

  • MIME type validation using wp_check_filetype()
  • File size limits enforced (10MB per file)
  • Rate limiting (5 files per session)
  • WordPress media library integration
  • Automatic cleanup after wizard completion

API Communication

The plugin communicates with the Synaplan backend API using WordPress HTTP API (wp_remote_post()):

  • All requests use HTTPS
  • Timeout set to 30 seconds
  • API key authentication
  • JSON request/response format
  • Error handling with fallback messages

Database Schema

The plugin stores configuration in WordPress options table:

Option Name Description
synaplan_wp_version Plugin version number
synaplan_wp_setup_completed Boolean flag for setup completion
synaplan_wp_api_key Encrypted API key
synaplan_wp_user_id Synaplan user ID
synaplan_wp_widget_config Widget configuration (JSON)

Hooks & Filters

The plugin provides hooks for developers to extend functionality:

  • synaplan_wp_before_widget_render - Before widget HTML is rendered
  • synaplan_wp_after_wizard_complete - After wizard completion
  • synaplan_wp_api_request_args - Modify API request arguments
  • synaplan_wp_widget_config - Filter widget configuration

üîß Troubleshooting

Common Issues & Solutions

‚ùå Problem: Plugin won't activate

Solution:

  • Check your PHP version (must be 7.3 or higher)
  • Ensure you have sufficient memory (128MB minimum)
  • Check for conflicting plugins
  • Try deactivating other plugins temporarily

‚ùå Problem: Wizard setup fails at Step 1

Solution:

  • Verify your password meets requirements (6+ chars, numbers, special characters)
  • Check that your email address is valid
  • Ensure you've accepted the Terms of Service
  • Check your internet connection

‚ùå Problem: File upload fails

Solution:

  • Check file size (max 10MB per file)
  • Verify file format (PDF, DOC, DOCX, TXT only)
  • Ensure you're not uploading more than 5 files
  • Check server upload limits in php.ini
  • Try uploading files one at a time

‚ùå Problem: Chat widget not showing on website

Solution:

  • Verify setup wizard was completed successfully
  • Clear your browser cache and WordPress cache
  • Check that the plugin is activated
  • Look for JavaScript errors in browser console (F12)
  • Ensure your theme's footer.php has wp_footer() call

‚ùå Problem: API connection errors

Solution:

  • Check your server's firewall settings
  • Ensure HTTPS is enabled on your site
  • Verify allow_url_fopen is enabled in PHP
  • Check for proxy or CDN issues
  • Contact your hosting provider if issues persist

Debug Mode

Enable Debug Mode in the wizard (Step 1) to see detailed console logs. This helps identify issues with:

  • Data collection and sanitization
  • API request/response
  • File upload process
  • Widget initialization
üí° Accessing Debug Logs: Open your browser's Developer Console (F12), go to the "Console" tab, and look for messages prefixed with [Synaplan]

Manual Cleanup

If you need to completely reset the plugin, you can use the manual cleanup function: 

// In WordPress admin, go to Tools > Site Health > Debug
// Or add this to your theme's functions.php temporarily:
add_action('admin_init', 'synaplan_wp_manual_cleanup');
‚ö†Ô∏è Warning: Manual cleanup will delete all plugin data, including your API key and widget configuration. You'll need to run the setup wizard again.
